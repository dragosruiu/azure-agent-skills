# Azure Container Instances (ACI)

## When to use this skill

- Quick / burst / batch container runs that don't need an orchestrator.
- Per-job isolated container groups (Linux multi-container, Windows
  single-container).
- Confidential containers on AMD SEV-SNP for sensitive workloads.

## When NOT to use this skill

- Long-running services or autoscale workloads — use
  [`azure-container-apps`](azure-container-apps/SKILL.md) or
  [`azure-aks-cluster`](azure-aks-cluster/SKILL.md).
- Per-PR / dev environment provisioning — use
  [`azure-deployment-environments`](../devops/azure-deployment-environments/SKILL.md).

## Container group basics

A **container group** = one or more containers scheduled on the same
host, sharing lifecycle / network / storage (= K8s Pod).

| Property | Detail |
| --- | --- |
| Multi-container | **Linux only**; Windows = single container per group |
| Resource floor | 1 CPU + 1 GB RAM per group; sum of container `requests` |
| Restart policy | `Always` (default), `OnFailure`, `Never` |
| Init containers | Linux; sequential before app containers; max 59; API ≥ `2019-12-01` |
| Volumes | Azure Files, `emptyDir`, `secret`, `gitRepo` |
| Volume mount in multi-container group | **Linux only** |
| Max image size | 15 GB |
| Probes | liveness + readiness (HTTP / exec) |

**DNS name label** for public groups: `{label}.{region}.azurecontainer.io`.
`autoGeneratedDomainNameLabelScope`: `Unsecure` (default; subdomain
takeover risk), `TenantReuse`, `SubscriptionReuse`, `ResourceGroupReuse`.

**VNet integration:** subnet must be **delegated** to
`Microsoft.ContainerInstance/containerGroups` and the container group
must be in the **same region** as the subnet. Subnet ≥ **/28**;
**network profiles are retired** as of API `2021-07-01`.

## Secure defaults

| Setting | Value | Why |
| --- | --- | --- |
| ACR pull credentials | **user-assigned managed identity** with `AcrPull` on the registry (`imageRegistryCredentials.identity`) | Admin user is risky; MI is rotation-free. **Windows containers only support UAMI**, not system-assigned. |
| ACR with PE | enable Trusted Services on the ACR | Otherwise the pull is blocked. |
| OS | Linux for multi-container | Windows is single-container. |
| `restartPolicy` | `OnFailure` (or `Never` for batch) | `Always` for daemons; pick deliberately. |
| `autoGeneratedDomainNameLabelScope` | `TenantReuse` (or stricter) on public groups | Prevents subdomain takeover when the group is deleted + recreated. |
| Secret env vars | `secureValue` (not `value`) | `value` shows up in logs / API responses. |
| TLS | TLS 1.2 enforced for all HTTPS | TLS 1.0/1.1 are retired. |
| Privileged containers | not allowed by ACI | n/a |
| Subnet delegation | `Microsoft.ContainerInstance/containerGroups` | Required before deploying. |
| Subnet size | **≥ /28** (avoid /29 — too few IPs → stuck containers) | Buffer for restart / scale. |
| Logs | `diagnostics.logAnalytics` to a LAW (or `azureMonitor`) | ACI doesn't retain logs long-term. |

## Recipe — Azure CLI (VNet-integrated, MI-based ACR pull, LAW logs)

```bash
RG=aci-rg
LOC=eastus
ACR=myacrregistry         # existing
LAW=myloganalytics        # existing
MI=mi-aci-pull
VNET=aci-vnet
SUBNET=aci-subnet
CG=mycg

az group create -n "$RG" -l "$LOC"

# 1. UAMI + acrpull
az identity create -g "$RG" -n "$MI"
MI_ID=$(az identity show -g "$RG" -n "$MI" --query id -o tsv)
MI_PRINC=$(az identity show -g "$RG" -n "$MI" --query principalId -o tsv)
ACR_ID=$(az acr show -n "$ACR" --query id -o tsv)
az role assignment create --assignee "$MI_PRINC" --scope "$ACR_ID" --role acrpull

# 2. VNet + delegated subnet
az network vnet create -g "$RG" -n "$VNET" -l "$LOC" \
  --address-prefix 10.0.0.0/16
az network vnet subnet create -g "$RG" --vnet-name "$VNET" -n "$SUBNET" \
  --address-prefix 10.0.0.0/24 \
  --delegations Microsoft.ContainerInstance/containerGroups

# 3. LAW credentials
LAW_ID=$(az monitor log-analytics workspace show -g "$RG" --workspace-name "$LAW" --query customerId -o tsv)
LAW_KEY=$(az monitor log-analytics workspace get-shared-keys -g "$RG" --workspace-name "$LAW" --query primarySharedKey -o tsv)

# 4. Container group
az container create -g "$RG" -n "$CG" -l "$LOC" \
  --image "$ACR.azurecr.io/myapp:1.0.0" \
  --acr-identity "$MI_ID" --assign-identity "$MI_ID" \
  --cpu 1 --memory 1.5 --os-type Linux \
  --restart-policy OnFailure \
  --vnet "$VNET" --subnet "$SUBNET" \
  --ports 80 \
  --environment-variables APP_ENV=production \
  --secure-environment-variables DB_PASSWORD=verysecret \
  --log-analytics-workspace "$LAW_ID" \
  --log-analytics-workspace-key "$LAW_KEY"

az container show -g "$RG" -n "$CG" --query "{state:instanceView.state, ip:ipAddress.ip}" -o table
az container logs -g "$RG" -n "$CG"
```

## Recipe — Bicep

```bicep
param location string = resourceGroup().location
param managedIdentityId string
param acrLoginServer string
param subnetId string
param logAnalyticsWorkspaceId string
@secure() param logAnalyticsWorkspaceKey string

resource cg 'Microsoft.ContainerInstance/containerGroups@2025-09-01' = {
  name: 'mycontainergroup'
  location: location
  identity: {
    type: 'UserAssigned'
    userAssignedIdentities: { '${managedIdentityId}': {} }
  }
  properties: {
    osType: 'Linux'
    restartPolicy: 'OnFailure'
    sku: 'Standard'                              // 'Standard' | 'Confidential' | 'Dedicated'

    imageRegistryCredentials: [
      {
        server: acrLoginServer
        identity: managedIdentityId              // MI-based ACR pull (no admin user)
      }
    ]

    subnetIds: [ { id: subnetId } ]              // subnet MUST be delegated

    ipAddress: {
      type: 'Private'                            // or 'Public' + dnsNameLabel + autoGenerated...Scope
      ports: [ { port: 80, protocol: 'TCP' } ]
    }

    diagnostics: {
      logAnalytics: {
        workspaceId: logAnalyticsWorkspaceId
        workspaceKey: logAnalyticsWorkspaceKey
      }
    }

    containers: [
      {
        name: 'myapp'
        properties: {
          image: '${acrLoginServer}/myapp:1.0.0'
          resources: {
            requests: { cpu: 1, memoryInGB: 1 }
            limits:   { cpu: 2, memoryInGB: 2 }
          }
          ports: [ { port: 80, protocol: 'TCP' } ]
          environmentVariables: [
            { name: 'APP_ENV',     value: 'production' }
            { name: 'DB_PASSWORD', secureValue: 'verysecret' }   // secureValue, not value
          ]
          livenessProbe: {
            httpGet: { path: '/health', port: 80, scheme: 'http' }
            initialDelaySeconds: 30, periodSeconds: 10
            failureThreshold: 3,    timeoutSeconds: 5
          }
          readinessProbe: {
            httpGet: { path: '/ready', port: 80, scheme: 'http' }
            initialDelaySeconds: 10, periodSeconds: 5
            failureThreshold: 3,    timeoutSeconds: 3
          }
        }
      }
    ]
  }
}
```

## Common failures

| Symptom | Cause | Fix |
| --- | --- | --- |
| Deploy fails: subnet not delegated | Subnet missing `Microsoft.ContainerInstance/containerGroups` delegation | `az network vnet subnet update --delegations Microsoft.ContainerInstance/containerGroups` |
| Region-mismatch error | VNet subnet and container group in different regions | Co-locate region |
| Multi-container fails on Windows | Windows ACI = single container only | Linux for multi-container; or split into separate single-container groups |
| ACR image pull fails (private endpoint ACR) | "Trusted services" not enabled on ACR; or admin user disabled with no MI | Enable Trusted Services; use UAMI + `AcrPull`; Windows = UAMI required (system MI not supported for ACR pull) |
| Logs disappear after container exits | ACI doesn't retain logs long-term | Configure `diagnostics.logAnalytics` to a LAW |
| Public IP changes on restart | ACI may rotate IPs (incl. on platform maintenance) | Front with App Gateway / Front Door for stable ingress; rely on FQDN, not IP |
| Subdomain takeover risk | `autoGeneratedDomainNameLabelScope` defaults to `Unsecure` | Set `TenantReuse` (or stricter) |
| Image > 15 GB → strange failures | exceeds ACI image limit | Multi-stage build, slim base, or split |
| Windows VNet container can't fetch MI tokens | IMDS `169.254.169.254` not available for Windows ACI in a VNet | Use `IDENTITY_ENDPOINT` + `IDENTITY_HEADER` env vars injected by ACI |
| Init container failure stops the group | `restartPolicy: 'Never'` on a group with init containers | Use `OnFailure`; check init logs separately |
| Old guides about network profiles fail | Network profiles **retired** in API `2021-07-01`+ | Use `subnetIds` directly |
| Stuck / unresponsive containers in /29 subnet | Only 3 usable IPs; ACI needs buffer | Use **/28** or larger |
